## 二、整体安全加固分层框架
1. **控制平面加固**：apiserver、etcd、controller-manager、scheduler认证、证书、加密、准入控制
2. **节点层加固**：操作系统内核、SSH、防火墙、SELinux、权限、无关服务关停
3. **容器运行时加固**：containerd、Pod安全上下文、PSS准入、特权能力限制
4. **镜像供应链安全**：私有仓库、漏洞扫描、标签规范、禁止公网镜像
5. **数据加密加固**：etcd静态加密、Secret管控、存储卷备份、传输TLS加密
6. **网络边界加固**：NetworkPolicy、Ingress白名单、出站访问控制
7. **运维管控加固**：RBAC最小权限、审计日志、证书轮换、密钥轮换

## 三、集群控制平面加固清单（PROD强制）
1. apiserver 关闭匿名访问、禁用基础认证/静态Token，仅保留X509证书+OIDC+ServiceAccount认证
2. 开启RBAC授权器，禁用ABAC、AlwaysAllow等不安全模式
3. 完整启用准入控制器链：PodSecurity、LimitRanger、ResourceQuota、Webhook校验
4. etcd 开启TLS双向加密通信，开启etcd静态加密存储Secret
5. 控制平面组件仅监听内网地址，禁止0.0.0.0公网暴露6443端口
6. 组件证书统一90天有效期，提前7天自动轮换，杜绝永久证书
7. 开启完整分级审计日志，高危操作实时告警，日志留存90天
8. 禁用默认全局ClusterAdmin账号长期绑定，仅应急临时使用并及时回收
9. kube-system命名空间严格限制操作权限，普通运维无删除权限
10. 所有集群资源变更纳入Git版本管理，高危操作双人复核

## 四、Linux节点操作系统加固规范
### 4.1 系统基础加固
1. 操作系统定期补丁更新，高危内核漏洞72小时内修复
2. 关闭SSH密码登录，仅允许密钥登录，禁用root远程SSH
3. SSH端口修改非22默认端口，IP白名单限制SSH登录源地址
4. 启用SELinux强制模式，禁止关闭SELinux
5. 防火墙放行最小端口集：6443、2049、111、53、节点通信端口，关闭多余端口
6. 卸载无用系统服务、禁用开机自启（FTP、Telnet、Apache等）
7. 系统关键目录权限加固 `/etc/kubernetes/ssl chmod 600`
8. 节点定时清理无用镜像、停止闲置容器，防止磁盘耗尽

### 4.2 容器运行时 containerd 加固
1. 禁用docker socket直接挂载Pod，仅使用标准CRI接口
2. containerd 配置镜像拉取白名单，仅允许内网Harbor地址
3. 关闭容器镜像特权挂载、限制宿主机目录hostPath挂载范围
4. 容器日志驱动统一json，禁止日志输出明文密钥
5. 限制容器最大并发、最大PID，防止容器DoS攻击节点
6. 禁用容器访问宿主机内核模块、sysctl修改高危参数

## 五、容器运行时安全加固（PSS Restricted标准）
### 5.1 Pod全局强制约束（准入Webhook拦截违规Pod）
1. `runAsNonRoot: true` 禁止root用户运行业务容器
2. `allowPrivilegeEscalation: false` 关闭权限提升
3. `capabilities.drop: ["ALL"]` 删除全部Linux特权能力
4. `readOnlyRootFilesystem: true` 只读根文件系统，临时数据使用emptyDir
5. seccompProfile启用RuntimeDefault系统调用过滤
6. 禁止 `privileged: true`、hostNetwork/hostPID/hostIPC共享宿主机命名空间
7. 禁止直接挂载宿主机敏感目录 `/etc` `/var/lib/kubernetes` `/root`

### 5.2 资源防护
1. 所有容器强制配置CPU/memory requests & limits，LimitRanger准入拦截缺失配置Pod
2. 设置 `terminationGracePeriodSeconds` 合理优雅关闭时间，避免强制杀进程丢失数据
3. 禁止无限制进程PID，设置`pids.limit`防进程耗尽攻击

## 六、镜像供应链安全加固规范
1. 统一内网私有Harbor仓库，禁止直接拉取docker.io公网镜像
2. 禁用`latest`浮动标签，所有镜像必须使用固定版本Hash/数字Tag
3. 镜像开启自动漏洞扫描，高危漏洞（Critical/High）镜像禁止部署UAT/PROD
4. 镜像构建禁止内置账号密码、密钥、证书，运行时通过Secret注入
5. 镜像分层精简，删除包管理器缓存、调试工具（curl、bash可按需最小化）
6. 镜像拉取统一使用imagePullSecrets，禁止匿名拉取私有仓库
7. 淘汰老旧、无维护镜像，定期清理Harbor过期无用镜像释放存储

## 七、全链路数据加密加固规范
### 7.1 传输加密
1. apiserver、etcd、kubelet通信全部TLS双向加密
2. Ingress全局强制HTTPS，HTTP跳转443加密访问
3. 内部数据库、RPC服务通信启用TLS加密传输

### 7.2 静态持久加密
1. PROD集群etcd开启静态加密，保护Secret资源
2. 数据库持久存储使用SSD分布式加密存储池
3. 备份文件自动加密存储，禁止明文备份业务数据
4. 集群CA私钥离线加密介质备份，双人保管，不存放集群节点

### 7.3 密钥生命周期管控
1. ServiceAccount Token、数据库密码、镜像仓库密钥按月轮换
2. 证书统一90天有效期，到期前7天轮换
3. 离职人员立即吊销证书、回收kubeconfig、OIDC账号禁用

## 八、网络边界安全加固
1. UAT/PROD所有Namespace默认拒绝入站流量，采用白名单NetworkPolicy
2. 网段隔离：FAT Pod网段禁止主动访问UAT/PROD业务网段
3. 中间件（MySQL/Redis）仅放行对应业务Pod端口，禁止全局开放3306/6379
4. 出站仅放行内网机房DNS、Harbor、业务依赖服务，阻断公网无限制出站
5. Ingress网关单独白名单管控，限制外部访问源IP段，CC限流防护
6. 53 DNS端口全局统一放行，不拦截域名解析流量

## 九、安全应急处置标准流程
### 9.1 高危安全事件分级
- 一级（紧急）：密钥泄露、越权删除生产数据、漏洞容器大规模部署、FAT直连生产数据库
- 二级（风险）：RBAC权限异常、证书过期、大量403越权访问、镜像高危漏洞
- 三级（一般）：违规Pod创建、临时策略配置、闲置高权限账号

### 9.2 一级事件应急SOP
1. 立即阻断风险流量：Ingress下线路由、暂停备份CronJob、隔离违规Namespace
2. 回收泄露密钥/证书，快速轮换所有相关凭证
3. 审计日志溯源操作人、操作时间、影响资源范围
4. 临时收紧网络策略，阻断跨环境非法访问
5. 数据备份完成后执行故障回滚，恢复业务
6. 24小时内输出安全事件复盘报告，优化加固规则

### 9.3 权限泄露应急
1. 立即删除对应ClusterRoleBinding/RoleBinding，回收账号权限
2. 吊销客户端证书、重置OIDC账号密码
3. 轮换所有被泄露的数据库、仓库密钥
4. 全集群审计日志检索该账号所有历史操作，排查数据篡改风险

## 十、DEV/FAT/UAT/PROD 加固差异化基线
### DEV本地
简化加固，仅本地自用，无强制加密、审计、网络隔离要求，仅作开发调试。
### FAT测试
1. 节点基础防火墙加固，无需严格SELinux强制模式；
2. PSS Baseline安全标准，不强制Restricted；
3. 镜像仅告警高危漏洞，不拦截部署；
4. 审计日志短期留存7天，无短信告警。
### UAT预生产
1. 加固标准完全对齐生产，仅开放少量测试只读权限；
2. 全量etcd加密、完整审计日志留存90天；
3. PSS Restricted全局生效，拦截特权容器；
4. 镜像高危漏洞直接拒绝部署。
### PROD生产（强制红线）
1. 控制平面、节点、容器、镜像、存储、网络全维度完整加固；
2. 最小权限RBAC、全局默认拒绝网络策略、完整加密体系；
3. 高危安全事件分级告警+短信通知，24小时运维响应；
4. 每月完整安全巡检，季度全集群漏洞扫描；
5. 所有安全变更双人复核，操作全程审计留痕。

## 十一、定期安全巡检标准化SOP
### 1. 每日巡检（自动化告警）
1. 审计日志403越权访问、删除资源、Secret变更异常告警
2. CSI/Calico/Ingress/CoreDNS系统组件异常Pod告警
3. 存储池磁盘满、备份任务执行失败告警
4. 证书到期7天内预警

### 2. 每周人工巡检
1. 清理闲置ClusterRoleBinding、冗余高权限账号
2. 清理过期快照、废弃PVC、无用Secret
3. 检查NetworkPolicy是否存在全开放高危策略
4. 核查镜像扫描高危漏洞列表，推动业务修复

### 3. 月度完整安全巡检（输出巡检报告）
1. 集群证书有效期全面核查，完成到期轮换
2. RBAC权限梳理，回收离职、闲置账号权限
3. 节点系统补丁、内核漏洞扫描修复
4. 准入控制规则校验，确认无旁路安全策略
5. 审计日志复盘，梳理异常访问行为
6. 密钥轮换执行情况核对，补全逾期未轮换密钥

### 4. 季度深度安全演练
1. 数据备份恢复演练，验证密钥、快照可用性
2. 模拟密钥泄露、越权操作应急处置演练
3. 全集群镜像漏洞扫描，推动业务修复高危漏洞
4. 网络隔离策略验证，测试FAT访问生产阻断效果

## 十二、生产安全最佳实践
1. 全程遵循最小权限、深度防御、多环境隔离三大安全原则；
2. 区分控制平面、节点、容器、镜像、存储、网络六层分层加固；
3. 所有安全配置纳入Git版本管理，变更留痕可追溯；
4. 自动化告警+人工定期巡检结合，提前发现安全风险；
5. 建立密钥、证书固定轮换周期，杜绝长期静态凭证；
6. 完善安全事件分级应急处置流程，故障发生可快速止损；
7. 定期演练备份恢复与安全应急，避免灾备失效、处置慌乱。
