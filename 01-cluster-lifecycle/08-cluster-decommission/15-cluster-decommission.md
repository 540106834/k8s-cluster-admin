# Kubernetes 集群整机退役销毁规范（15\-cluster\-decommission\.md）Ubuntu22\.04 离线生产版

## 1\. 文档概述

### 1\.1 文档定位

本文为 Kubernetes 生产集群**整机退役、环境下线、资源清零、服务器回收、机房退服**标准化运维规范，完全适配 **Ubuntu22\.04、K8s v1\.32、kubeadm 自建离线 Harbor 集群**。作为集群生命周期最后一环，统一退役前置核验、业务割接、集群销毁、节点清零、数据销毁、归档闭环全流程，杜绝残留脏配置、数据泄露、资源残留、环境冲突等问题，为集群正式下线唯一合规落地基准。

### 1\.2 核心退役原则（生产红线）

- **先迁移后销毁**：所有业务、数据、配置完全迁移至新集群并稳定运行后，方可启动退役流程

- **数据优先保全**：退役前完成全量数据备份、快照留存、日志归档，杜绝数据丢失

- **分层有序销毁**：业务层→组件层→集群层→系统层逐层清零，禁止暴力直接重装系统

- **全程可追溯**：所有退役操作留痕、日志留存、资源核销、台账归档

- **清零彻底无残留**：集群配置、证书、容器数据、存储挂载、网络规则全部清空

- **回收合规安全**：服务器下线前完成数据彻底销毁，防止敏感数据泄露

### 1\.3 适用场景

- 机房搬迁、老旧硬件淘汰、服务器设备报废回收

- 旧版本集群整体替换、业务全量迁移至新标准化集群

- 测试/预发废弃环境、临时集群、过期演练环境清零销毁

- 集群基线彻底错乱、故障频发，无保留价值需要整机重建

- 项目下线、业务关停，配套K8s集群整体退服销毁

## 2\. 退役前置核验（强制必做）

未完成以下所有核验项，禁止执行任何集群销毁、节点清零操作。

### 2\.1 业务全量割接核验

- 所有业务流量100%切换至目标集群，旧集群无在线流量、无用户访问

- 有状态中间件、数据库、缓存数据完全同步，新旧数据一致无差异

- 业务持续稳定运行72小时以上，无报错、无抖动、无回滚记录

- CI/CD流水线、域名解析、负载均衡、监控告警全部切离旧集群

### 2\.2 全量数据备份归档

- 执行ETCD全量快照备份，留存最终时间点集群所有资源数据（参考`12-etcd-backup.md`）

- 备份所有PV持久化数据、业务挂载目录、数据库本地数据

- 归档集群证书、kubeconfig、系统组件配置、自定义脚本、运维台账

- 汇总集群运行日志、容器日志、系统日志，统一归档留存

### 2\.3 集群资源盘点确认

```bash
# 核查全量资源无遗漏
kubectl get all --all-namespaces
kubectl get pv,pvc,sc --all-namespaces
kubectl get configmaps,secrets --all-namespaces
kubectl get ingress,service --all-namespaces

```

确认无遗留核心业务、无未迁移私有配置、无独占存储资源。

### 2\.4 权限与流程审批

- 完成集群退役工单审批，确认下线原因、影响范围、回收方式

- 关闭所有集群外部访问权限、堡垒机权限、API访问入口

- 暂停所有定时备份、监控告警、自动运维任务

## 3\. 集群分层退役标准流程

严格遵循**业务销毁 → 组件停止 → 集群重置 → 节点清零 → 系统清理** 分层流程，保证环境彻底干净无残留。

### 3\.1 第一层：业务资源清空

批量清理所有业务负载与自定义资源，释放集群调度资源

```bash
# 删除所有业务命名空间（自动回收所有关联资源）
kubectl delete ns $(kubectl get ns | grep -v kube-system | grep -v default | awk '{print $1}')

# 核查残留资源
kubectl get all --all-namespaces

```

优先删除业务命名空间，自动级联回收Deployment、StatefulSet、Pod、Service、Ingress等资源，避免残留孤立资源。

### 3\.2 第二层：系统组件优雅停止

停止集群核心服务，防止销毁过程中自动重建资源

```bash
# 所有节点停止kubelet服务
systemctl stop kubelet
systemctl disable kubelet

# 清空容器运行时所有残留容器与镜像
crictl stop $(crictl ps -a -q)
crictl rm $(crictl ps -a -q)
crictl rmi $(crictl images -q)

```

### 3\.3 第三层：kubeadm 集群整机重置

Master节点执行集群重置，销毁控制面、ETCD、证书、集群配置

```bash
# 集群整机重置
kubeadm reset -f

# 清理残留网络与路由
rm -rf /etc/cni/net.d/*
ip link flush all
ip route flush all

```

**高可用集群操作规范**：逐台Master节点执行reset操作，确保所有控制面配置彻底清零。

### 3\.4 第四层：节点深度清零（生产关键）

清理所有集群残留目录、证书、配置、数据，恢复服务器基线环境

```bash
# 清理K8s核心目录
rm -rf /etc/kubernetes/*
rm -rf /var/lib/kubelet/*
rm -rf /var/lib/etcd/*
rm -rf /var/lib/cni/*

# 清理容器运行时残留
rm -rf /var/lib/containerd/*

# 清理本地kubeconfig
rm -rf /root/.kube/*
rm -rf $HOME/.kube/*

```

### 3\.5 第五层：系统环境复位

关闭开机自启、恢复系统默认配置，适配服务器回收或复用场景

```bash
# 关闭服务自启
systemctl disable containerd
systemctl disable kubelet

# 重启服务器彻底生效
reboot
```

## 4\. 存储数据专项销毁规范

针对持久化存储数据，根据设备用途区分**复用擦除**与**报废销毁**两种标准，杜绝敏感数据残留。

### 4\.1 服务器复用场景（机房留用）

- 逐磁盘格式化业务数据分区、清空PV挂载目录数据

- 覆盖写入空白数据擦除磁盘残留缓存与日志

- 校验目录无残留业务数据、密钥、配置文件

### 4\.2 服务器报废场景（设备退网）

- 执行磁盘全盘清零或专业数据粉碎操作

- 禁止仅格式化表层分区，杜绝数据恢复风险

- 出具数据销毁核验记录，归档机房设备台账

## 5\. 退役后核验标准（闭环必过）

### 5\.1 集群层清零核验

- 无任何K8s集群配置、证书、kubeconfig残留

- kubelet、containerd无残留运行进程、无自启任务

- CNI网络配置、路由、网卡规则完全清空

### 5\.2 资源层清零核验

- 无残留容器、镜像、存储挂载、临时资源文件

- 无ETCD数据、集群状态数据、业务缓存数据

- 无自定义脚本、定时任务、运维工具残留

### 5\.3 系统层核验

- 系统基线正常、网络正常、无异常后台进程

- 服务器可正常开机、重启、远程管理，满足复用/报废标准

## 6\. 退役归档与台账闭环

### 6\.1 归档内容清单

- 集群最终ETCD快照、全量资源YAML备份

- 退役操作日志、操作记录、变更工单

- 设备信息、机房机位、资产编号、下线时间

- 数据销毁核验报告、业务割接确认单

### 6\.2 台账核销

- 更新机房资产台账，标记设备退役/回收状态

- 注销集群运维权限、监控面板、域名解析、仓库授权

- 关闭集群相关自动化运维、备份、告警策略

## 7\. 退役异常处理方案

- **资源删除卡住**：针对Terminating残留资源，强制删除finalizers后置清理

- **目录权限残留**：递归修改目录权限后强制清空残留文件

- **服务无法停止**：kill强制终止进程，禁用自启后重启机器清零

- **网络残留异常**：重启网络服务、刷新路由规则，重置CNI配置

## 8\. 生产退役红线规范

- 禁止业务未完全迁移、流量未切离直接销毁集群

- 禁止未备份数据、未归档台账直接下线设备

- 禁止暴力重装系统替代规范清零，造成资产记录混乱

- 禁止存储数据未彻底销毁直接报废服务器，引发数据泄露

- 禁止未完成全维度核验，直接结束退役流程

- 禁止退役后保留集群残留配置、权限、自动化任务

- 禁止业务高峰期、系统不稳定期间执行集群退役销毁操作
