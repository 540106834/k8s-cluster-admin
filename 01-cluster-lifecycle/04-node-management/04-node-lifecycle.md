2\. 生产环境标准维护窗口规范


所有节点生命周期变更操作（升级、重启、替换、变更配置），仅允许在**低峰维护窗口**执行

### 
2\.1 集群标准化维护窗口


- 日常维护窗口：周一\~周五 00:00\-04:00（业务最低峰）


- 重大变更窗口：周末 01:00\-05:00，同步运维值班人员在岗

- 紧急故障变更：不受窗口限制，走应急Runbook（10\-runbooks\.md）


### 
2\.2 维护窗口操作前置审批


- 常规变更：运维内部工单审批备案


- 节点升级/整机替换：业务\+运维双部门审批，预留回滚预案


- 窗口内操作必须留存：操作日志、节点变更前后快照


### 2\.3 维护窗口操作标准流程


工单审批 → 维护窗口开启 → 节点Cordon隔离 → Drain迁移业务 → 生命周期变更 → 健康校验 → Uncordon恢复 → 工单闭环

## 
3\. Worker 节点集群版本升级操作


控制面版本固定，仅升级Worker侧 kubeadm/kubelet/kubectl 组件；**单节点灰度升级，禁止批量并行升级**

### 
3\.1 升级前置准备


1. 维护窗口内执行，完成节点 `cordon` 调度隔离


2. 执行标准drain迁移节点全部业务Pod（参考 03\-node\-drain\.md）


3. 节点本地备份 /etc/kubernetes、/var/lib/kubelet 配置目录


### 
3\.2 分步升级流程（Worker节点本地执行）


```bash
# 1. 更新apt源，安装目标版本组件
apt update && apt install -y kubeadm=1.32.13-00 kubelet=1.32.13-00 kubectl=1.32.13-00

# 2. 升级节点kubelet配置
kubeadm upgrade node

# 3. 重启kubelet生效
systemctl daemon-reload
systemctl restart kubelet

# 4. 校验节点状态
kubectl get nodes

# 5. 升级完成解除调度隔离
kubectl uncordon <node-name>
```

### 
3\.3 批量节点升级管控规范

- 严格串行执行：单节点升级完成Ready后，再操作下一台Worker


- 同批次集群内最多同时一台节点变更，避免业务大面积迁移


- 升级异常立即执行回滚：降级组件版本 \+ kubeadm reset恢复配置

## 
4\. 服务器整机节点替换流程


适用场景：服务器硬件老化、磁盘故障、机型扩容替换、机房机器置换

### 
4\.1 整体流程链路


旧节点隔离驱逐 → 新节点环境初始化 → 新节点接入集群 → 业务灰度切换 → 旧节点集群剔除 → 硬件回收

### 
4\.2 分步标准化操作


1. **旧节点下线**：参照 03\-node\-drain\.md 执行 cordon \& drain 迁移全部业务


2. **新节点初始化**：系统环境、containerd、内核参数、时间同步基线配置


3. **新节点入网**：参照 02\-node\-join\.md 执行 kubeadm join 接入集群


4. **业务灰度迁移**：通过节点标签/调度亲和，分批引导业务调度至新节点


5. **下线旧节点**：业务全部迁移完成，删除旧节点集群资源，机器下架回收


### 
4\.3 节点替换关键管控点


- 保留原节点标签、污点、资源配额配置，业务调度逻辑无变更


- 存储PV数据跨节点正常挂载，校验静态/动态存储连通性


- 替换全程业务零中断，禁止大批量业务同时迁移

## 
5\. 业务无感知节点灰度迁移方案


适用：集群节点资源重分配、业务分区调度、机房业务流量迁移，**不中断在线业务**

### 
5\.1 灰度迁移核心策略


不强制驱逐Pod，通过调度权重\+节点标签，等待业务滚动重建自然完成节点迁移

### 
5\.2 实操步骤


```bash
# 1. 目标节点设置调度冻结（不驱逐存量运行Pod）
kubectl cordon k8s-worker-01

# 2. 业务分批滚动重启，自然调度至其他正常节点
kubectl rollout restart deployment <biz-deploy-name> -n production

# 3. 确认存量Pod全部迁移后，执行节点drain清理残留容器
kubectl drain k8s-worker-01 --ignore-daemonsets --delete-emptydir-data

# 4. 变更完成按需恢复调度或剔除节点
kubectl uncordon k8s-worker-01
```

### 
5\.3 灰度迁移适用场景


- 生产高峰温和迁移业务，拒绝突发流量抖动


- 大批量集群节点轮换维护


- 核心高可用在线业务迁移

## 
6\. 节点生命周期异常兜底\&回滚流程


### 
6\.1 升级失败紧急回滚


```bash
# 1. 故障节点紧急隔离，阻断业务调度
kubectl cordon <failed-node>

# 2. 降级k8s组件至原稳定基线版本
apt install kubeadm=1.32.12-00 kubelet=1.32.12-00 --downgrade

# 3. 重置节点异常配置，重启服务
systemctl restart kubelet containerd

# 4. 校验节点Ready状态，恢复业务调度
kubectl uncordon <failed-node>
```

### 
6\.2 节点替换失败兜底


- 新节点接入异常：直接清空配置 kubeadm reset，复用旧节点承接业务


- 业务迁移异常：解除旧节点cordon，回流全部业务流量
