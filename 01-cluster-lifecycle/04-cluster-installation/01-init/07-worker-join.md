## 四、Master节点操作：生成节点加入命令
### 4.1 查看当前有效join token（默认有效期24h）
```bash
kubeadm token list
root@k8s-master-192-168-11-161:~# kubeadm  token list

i7q4pw.rh8q88pmywd6dfck   13h         2026-07-04T17:40:48Z   authentication,signing   The default bootstrap token generated by 'kubeadm init'.   system:bootstrappers:kubeadm:default-node-token

```
### 4.2 Token过期，重新生成带hash的完整join命令
```bash
kubeadm token create --print-join-command
```
输出示例（复制保存，用于worker节点执行）：
```
kubeadm join 192.168.11.161:6443 --token abcdef.0123456789abcdef \
  --discovery-token-ca-cert-hash sha256:xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

### 4.3 备用：仅已知hash，手动创建临时token
```bash
# 创建24小时有效期token
kubeadm token create --ttl 24h
# 获取CA hash
openssl x509 -pubkey -in /etc/kubernetes/pki/ca.crt | openssl rsa -pubin -outform der 2>/dev/null | openssl dgst -sha256 -hex | sed 's/^.* //'
```

## 五、Worker节点执行kubeadm join加入集群
### 5.1 完整加入命令（复制Master输出的命令直接执行）
```bash
kubeadm join 192.168.11.161:6443 --token abcdef.0123456789abcdef \
  --discovery-token-ca-cert-hash sha256:xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx \
  --cri-socket=unix:///run/containerd/containerd.sock \
  --ignore-preflight-errors=NumCPU,Mem
```

### 参数说明
1. `--cri-socket`：指定containerd运行时套接字，与集群统一
2. `--ignore-preflight-errors`：低配机器忽略CPU、内存预检报错
3. token过期/哈希错误，回到Master重新生成join命令

## 十一、常见故障排查
1. **kubeadm join 校验CA hash失败**
   Master重新执行 `kubeadm token create --print-join-command` 获取最新hash
2. **token已过期**
   Master重新生成长期token，重新执行join
3. **节点加入后状态NotReady**
   检查calico-node Pod日志、确认CNI插件/opt/cni/bin完整
4. **kubelet启动报错cgroup不匹配**
   核对containerd与kubelet均为systemd cgroup驱动
5. **镜像拉取ImagePullBackOff**
   检查02-containerd.md中harbor insecure_skip_verify配置

## 十二、补充：部署 metrics-server 解决 kubectl top 不可用
### 12.1 问题说明
执行 `kubectl top nodes` 提示 `error: Metrics API not available`，原因集群未部署指标采集组件 metrics-server。

### 12.2 下载并修改部署清单
```bash
cd /usr/local/src
wget https://github.com/kubernetes-sigs/metrics-server/releases/download/v0.7.2/components.yaml

# 替换镜像为内网Harbor
sed -i 's|registry.k8s.io/metrics-server|harbor.jinshaoyong.com/k8s|g' components.yaml
# 在 args: 增加跳过kubelet证书校验参数
  - --kubelet-insecure-tls
```

### 12.3 部署 metrics-server
```bash
kubectl apply -f components.yaml
```
