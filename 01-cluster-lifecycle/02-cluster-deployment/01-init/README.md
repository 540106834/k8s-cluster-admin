### 1.1 全局固定组件版本（全集群统一，禁止自定义变更）

本次集群部署采用**稳定分支定点版本选型**，优先选择K8s官方维护稳定版、组件跨版本兼容认证通过版本，适配Ubuntu22.04系统生产环境，全套组件版本锁定如下：

- **底层操作系统**：Ubuntu 22.04 LTS x86_64（长期支持LTS版本，5年系统安全维护）
- **Kubernetes 集群版本**：**v1.32.13**（修正：官方稳定补丁版，生产首选；修复v1.32.x早期高危漏洞，纳入K8s官方长期补丁维护队列）
- **CRI容器运行时**：containerd v2.1.2（K8s 1.32官方认证标准CRI运行时）
- **容器底层执行引擎**：runc v1.1.15（适配containerd v2.1.x稳定分支，无运行时崩溃已知BUG）
- **集群网络CNI插件**：Project Calico v3.29.0（适配K8s 1.32全版本，BGP模式生产稳定最优版本）
- **企业私有镜像仓库**：Harbor v2.11.0（兼容1.32集群全部基础镜像、无CRI鉴权兼容问题）
- **集群运维工具**：crictl v1.32.0（与K8s大版本强对齐，CRI接口完全适配）
- **系统基础依赖服务**：chrony 系统默认版本（全节点时钟同步）

### 1.2 版本选型推进逻辑（生产环境落地标准）

#### 1.2.1 K8s v1.32.13 版本推进原因

1. **安全合规推进**：摒弃早期v1.32.0/v1.32.2低版本，修复K8s apiserver、etcd、kubelet高危权限绕过、拒绝服务高危CVE漏洞，满足生产环境安全基线准入要求
2. **生命周期推进**：K8s 1.32为官方标准稳定分支，v1.32.13是该分支累计补丁最全、线上故障最少的终期稳定子版本；集群后续仅做小版本补丁升级，不跨大版本迁移
3. **生态兼容推进**：完美适配Calico、Harbor、containerd主流生态组件，无CRI接口废弃、资源API字段淘汰兼容性问题
4. **运维成本控制**：版本社区案例丰富，线上故障方案成熟，降低后续集群运维、故障排查难度

#### 1.2.2 Containerd v2.1.2 版本推进原因

1. **官方标准约束**：K8s社区已全面废弃Docker运行时，强制推进CRI标准containerd作为唯一容器运行时
2. **版本兼容性推进**：v2.1.2专门适配K8s 1.30~1.34全分支版本，cgroup驱动、sandbox镜像、CRI GRPC接口和v1.32.13完全兼容
3. **性能生产推进**：修复高并发Pod场景下容器僵死、镜像拉取超时、socket权限异常问题，满足中小型生产集群负载压力

#### 1.2.3 周边组件版本对齐推进原则

- **大版本对齐**：crictl 工具严格和K8s集群大版本对齐（v1.32.x），保证CRI调用链路无异常
- **生态适配对齐**：Calico、Harbor选用对应时间段主流稳定版，跟随K8s主线版本做兼容性适配，不选用最新内测版
- **底层依赖锁定**：runc版本跟随containerd主线版本锁定，不单独升级底层运行引擎，规避容器启动异常

### 1.3 后续集群版本迭代推进规划

1. **稳定运行周期**：集群落地后长期锁定 v1.32.13 版本运行，不主动跨大版本升级至1.33+
2. **补丁迭代推进**：仅接收1.32分支官方安全补丁，通过 **09-upgrade.md** 执行在线小版本灰度升级
3. **组件联动升级**：K8s小版本升级时，周边containerd/Calico/Harbor版本保持不变，无特殊漏洞不联动升级
4. **版本淘汰计划**：等待K8s 1.32分支停止维护后，整体批量迁移至下一LTS稳定分支

## 六、已知网络问题&前置规避方案（重点）

根据环境链路探测，当前环境存在**外网Github、部分官方镜像站点解析/下载失败**问题，部署前必读以下规避方案：

1. **Github资源下载失败**：containerd、runc、crictl、Harbor离线包Github链接无法访问；  
✅ 解决方案：提前离线下载对应版本二进制安装包，上传至服务器`/usr/local/src`目录本地安装
2. **K8s阿里云密钥页面解析失败**：阿里云K8s apt密钥网页无法访问；  
✅ 解决方案：使用国内备用K8s源密钥，或提前导入离线gpg密钥文件
3. **容器镜像加速器解析失败**：中科大、阿里云公有镜像加速域名访问异常；  
✅ 解决方案：注释公有加速器，优先使用**内网Harbor私有仓库**拉取基础镜像
4. 阿里云Ubuntu基础源正常可用，系统软件安装无阻碍
